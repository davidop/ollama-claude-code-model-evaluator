# v0.1.0 Release Notes

## Summary

First public release of Ollama Claude Code Model Evaluator.

This release focuses on practical local-model selection for coding tasks with reproducible benchmark outputs and direct Claude Code integration.

## Highlights

- Dependency-free benchmark CLI (Python standard library only)
- Standard benchmark flow for publishing comparable results
- Quality + speed scoring across coding-oriented prompts
- Interactive dashboard for local result exploration
- Hardware-oriented model recommendations
- Mobile execution guide (Android Termux -> local PC Ollama)
- DevContainer for consistent setup
- GitHub Actions validation workflow

## Benchmarks Included In This Release

- Standard benchmark: [results/benchmark-standard.json](../../results/benchmark-standard.json)
- Extended context benchmark (16384 + 14b): [results/benchmark-ctx16384-plus14b.json](../../results/benchmark-ctx16384-plus14b.json)

## How To Reproduce

1. Start Ollama locally.
2. Run standard benchmark:
   - Linux/macOS: bash scripts/run-basic.sh
   - Windows: scripts/run-basic.ps1
3. Optionally run extended benchmark:
   - python eval_ollama_models.py --pull --num-ctx 16384 --output ./results/benchmark-ctx16384-plus14b.json --models qwen2.5-coder:3b qwen2.5-coder:7b qwen2.5-coder:14b deepseek-coder:6.7b

## Release Assets

Attach these files to the GitHub release:

- results/benchmark-standard.json
- results/benchmark-ctx16384-plus14b.json
- dashboard.html (optional)

## Quality Guardrails

- `python scripts/validate-benchmark-json.py` validates benchmark JSON structure and score ranges.
- `python scripts/validate-markdown-links.py` checks local README links.
- `bash scripts/release-check.sh` runs the pre-release checklist end-to-end.

## Known Limitations

- Quality scoring is heuristic and prompt-keyword based
- Results are machine-dependent by design
- Throughput/latency can vary across repeated runs
